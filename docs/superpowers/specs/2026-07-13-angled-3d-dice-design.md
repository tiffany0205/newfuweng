# 斜置立体骰子与待机呼吸动效设计

## 目标

将棋盘中央等待点击的骰子从正面平视效果调整为真正可辨认的斜置六面体，并让待机、点击翻滚、结果停留三个状态保持连续一致。

## 当前问题

当前骰子已经由六个 CSS 立体面组成，但 `face-1` 的待机姿态为 `rotateX(0) rotateY(0)`，镜头只能看到正面，因此视觉上近似平面。现有浮动动画作用于整个按钮，提示文字也会随之移动，没有形成“骰子悬浮、文字稳定”的层次。

## 结构调整

- 在 `dice-stage` 内为 `dice-cube` 增加独立的 `dice-float-shell` 容器。
- `dice-float-shell` 只负责待机悬浮、轻微缩放和光泽呼吸。
- `dice-cube` 继续负责六个面的 3D 姿态、翻滚和最终点数朝向。
- `dice-shadow` 保持为浮动容器的同级元素，独立执行阴影收缩和透明度呼吸。
- 骰子提示文字不参与悬浮动画，保持稳定可读。

## 斜置立体姿态

- 默认姿态使用约 `rotateX(-18deg) rotateY(30deg) rotateZ(-2deg)`，同时看到主点数面、顶部和右侧。
- 六个结果类都在原有朝向上叠加统一斜置角，使最终点数仍是主视觉面，同时保留相邻两个面的厚度。
- 每个面增加细微边缘高光和分层明暗；不使用图片、Canvas、WebGL 或外部动画依赖。
- 透视距离保持适中，避免顶部或侧面过度拉伸。

## 待机呼吸

- 周期约 2.4 秒，使用平滑的 ease-in-out。
- 骰子本体在 `translateY(0)` 与 `translateY(-6px)` 之间缓慢悬浮，同时最大缩放不超过 `1.025`。
- 高光与投影强度轻微变化，不持续旋转，不改变当前点数面。
- 地面阴影与骰子反向配合：骰子上升时阴影略缩小、变淡，下降时恢复。
- 鼠标悬停只轻微提亮；按下时短暂缩小，点击区域保持原尺寸。

## 点击翻滚与停留

- 点击后暂停待机呼吸，骰子从当前斜置角度进入现有 820ms 翻滚序列。
- 翻滚只使用 `transform`，不改变布局。
- 服务端结果返回后，移除翻滚状态并切换对应 `face-N` 类。
- 最终状态使用对应点数的斜置 3D 姿态，不回到平视角度。
- 下一次等待期间，从结果姿态继续执行待机呼吸。

## 响应式和可访问性

- PC 保持约 104px 骰子尺寸，确保三个可见面具有明显厚度。
- 手机端保持约 54px 尺寸和同样斜置角度，适当降低位移幅度与阴影模糊，避免挤压中央信息。
- 键盘焦点沿用 `dice-stage` 的清晰轮廓。
- `prefers-reduced-motion: reduce` 开启时停止悬浮、阴影呼吸和翻滚动画，但保留静态斜置 3D 姿态和最终点数。

## 测试与验收

- Blade 中存在独立的 `dice-float-shell`，提示文字不位于该容器内。
- 六个 `face-N` 状态均包含非零 X/Y 斜置角，不再使用正面平视姿态。
- 待机动画作用于浮动容器而不是整个按钮。
- 翻滚期间待机动画暂停，结束后恢复。
- 阴影具有与浮动同步的呼吸动画。
- PC 和手机 CSS 构建通过，减少动态效果规则保留静态 3D 姿态。
- 不修改走棋接口、随机点数、音效、弹框或奖励逻辑。
