# 地标识别与全量掷骰反馈设计

## 目标

修复棋盘颜色语义和地标收集结算不一致的问题，并让每一次成功的掷骰操作都有明确、统一的视觉与声音反馈。

## 棋盘颜色规则

- 紫色只用于 `category = landmark` 的 12 个正式地标格。
- 正式地标格同时显示独立地标图标、地标名称和“地标”角标，PC 与手机端使用同一语义。
- 电池、VIP、现金和机会属于奖励格，统一使用金色奖励标识；电池不再使用紫色光条。
- 增益格使用绿色，风险格使用红色或冰蓝色，安全格使用中性灰色。
- 棋盘图例与实际 CSS 分类保持一一对应，不能再出现一种颜色表达两个分类的情况。

正式地标名称保持现有配置，不做改名：旅行驿站、阳光大道、彩虹街区、欢乐车站、幸运转角、星光广场、海滨大道、音乐小镇、森林公园、梦想港湾、金色大道和终点冲刺。

## 地标收集结算

### 直接落地

骰子直接落到地标时，沿用现有逻辑：首次到达写入地标记录并解锁印章，重复到达增加访问次数和幸运值。

### 位移后落地

前进或后退格改变棋子位置后，后端重新读取最终停留格：

- 如果最终格是地标，执行地标解锁或重复访问逻辑。
- 初始前进或后退效果仍然保留在结果说明中。
- 最终格若是电池、VIP、现金、机会、冰冻或炸弹，不再触发第二次效果，避免奖励连锁、风险连锁和无限移动。
- 同一次走棋最多记录一个地标访问，数据库唯一约束继续承担幂等保护。

这条规则覆盖目前会移动到阳光大道、星光广场、梦想港湾和金色大道的前进/后退格。

## 统一结果弹框

每次成功的走棋请求都显示同一个结果弹框组件，不再只为 VIP、电池、现金和新地标弹框。弹框固定包含：

- 本次骰子点数或“解冻完成”状态。
- 最终停留格名称。
- 完整结果说明，包括前进、后退、地标解锁、获得奖励或风险结果。
- 明确的关闭按钮；普通结果短暂停留后可以自动关闭，奖励和新地标等待用户确认。

弹框按结果使用四种语义外观，但布局、动效和操作保持一致：

| 类型 | 场景 | 视觉语义 |
| --- | --- | --- |
| 普通 | 安全抵达、普通解冻 | 中性蓝灰 |
| 增益/地标 | 前进、机会、地标访问 | 绿色或紫色 |
| 奖励 | 电池、VIP、现金奖品 | 金色 |
| 风险 | 后退、冰冻、炸弹 | 红色或冰蓝色 |

弹框进入使用 200–300ms 的淡入和轻微缩放，退出时间更短。`prefers-reduced-motion` 开启时只做即时淡入淡出，不使用位移、缩放或庆祝粒子。

## 统一声音反馈

- 每次成功走棋都播放一次短反馈音，不再存在静默落地。
- 使用浏览器 Web Audio API 生成本地短音，不依赖外部音频文件或第三方资源。
- 普通、地标/增益、奖励、风险使用不同但长度一致的音调；VIP、现金等大奖可以在奖励音调上增加一个短高音，但仍属于同一反馈体系。
- 音频仅由用户点击骰子后触发，符合浏览器自动播放限制。
- 棋盘提供声音开关，状态保存在 `localStorage`；关闭后所有走棋音效静音，视觉弹框不受影响。
- 无法创建 AudioContext 或设备不支持时静默降级，不能影响走棋、弹框或错误处理。

## 前端数据与状态

后端走棋响应增加明确的反馈字段，前端不再通过搜索中文结果文案判断是否中奖或解锁地标：

- `feedback_type`：`normal`、`boost`、`landmark`、`reward` 或 `risk`。
- `final_cell_label`：棋子最终停留格名称。
- `landmark_unlocked`：本次是否首次解锁地标。
- `landmark_count` 与 `landmark_total`：最新地标收集进度。

现有点数、圈数、位置和结果文案字段继续保留，避免破坏当前页面更新逻辑。

## 错误与边界处理

- 机会不足、活动未开始、网络失败等失败响应不播放成功音效，也不显示成功结果弹框；继续使用现有错误提示。
- 重复请求沿用 `request_id` 幂等机制，不能重复发奖励或重复增加地标访问次数。
- 重复访问已解锁地标只增加访问次数和幸运值，不增加“已收集地标数量”。
- 弹框显示期间禁止重复提交骰子；关闭后恢复操作。

## 测试范围

- 电池奖励格不再使用紫色地标样式，正式地标仍使用紫色并带地标角标。
- 直接落到新地标时进度增加。
- 前进到新地标时进度增加，但不触发目标格的其他奖励。
- 后退到新地标时进度增加。
- 重复地标不增加唯一收集数量。
- 每种成功走棋响应都返回完整反馈字段。
- 前端普通、地标、奖励和风险结果都调用统一弹框与声音反馈。
- 静音状态、Web Audio 降级及减少动态效果不会阻塞操作。

## 不在本次范围

- 不修改地标名称和现有地标奖励数值。
- 不增加外部音乐文件或背景音乐。
- 不让位移后的最终格触发地标以外的连锁奖励或惩罚。
