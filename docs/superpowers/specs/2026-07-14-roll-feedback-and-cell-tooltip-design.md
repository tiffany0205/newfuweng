# 掷骰结果层级与棋盘格提示设计

## 目标

让用户在一次掷骰结算后立即看懂“发生了什么、最终停在哪里、本次获得或失去了什么、结算后余额是多少”，并让 PC 用户能够在当前棋盘格附近清晰看到格子效果说明。

## 掷骰结果弹框

### 信息层级

弹框不再重复展示后端拼接的完整长句，改为四层结构：

1. 核心结果：最大标题，例如“后退 2 格”“返回本圈起点”“VIP 等级 +1”。
2. 最终落点：独立区域展示“第 N 格 · 格子名称”。
3. 本次结算：按来源逐行展示影响或收益，例如“重复到达地标 / 幸运值 +1”“梦想港湾效果 / 幸运值 +1”。
4. 结算后状态：仅在相关数值存在时展示，例如“当前幸运值：2”。

“+N”始终表示本次变化，“当前 N”始终表示结算后的总数，禁止用“获得 2 点”表达累计余额。

### 结构化数据

- 走棋响应新增 `result_summary`，包含 `headline`、`destination`、`items` 和 `balances`。
- `items` 每项包含稳定的 `kind`、用户可读 `label` 和 `value`，前端不解析 `result_text`。
- `result_text` 和数据库历史记录继续保留，供机会流水、历史兼容和排障使用。
- 地标结算明确返回本次幸运值变化；重复印章与地标自身效果分别成项。
- 炸弹、冰冻、前进、后退、普通格、VIP、电池、额外机会和现金奖品均映射到统一结构；没有额外收益时显示一条简洁的影响说明。
- 幂等重放必须生成与首次请求一致的结构化结果，不新增依赖临时前端状态的字段。

## 棋盘格提示

### PC 锚定浮层

- 鼠标或键盘聚焦格子时，在该格子附近显示宽 280px 左右的提示卡，并用方向箭头指向格子。
- 优先显示在格子上方；空间不足时显示在下方，靠近左右边缘时自动限制在视口内。
- 标题为“第 N 格 · 名称”，类型标签、效果说明和解锁/到访状态分层展示。
- 被查看格增加清晰描边和轻微放大；移出格子后延迟约 150ms 关闭，允许鼠标移动到提示卡内阅读和点击玩法链接。
- 提示卡使用 `position: fixed`，避免被棋盘容器裁切；滚动或窗口缩放时重新定位。

### 手机与触屏

- 点击格子后继续使用底部说明面板，不模拟 hover。
- 面板标题和正文采用与 PC 相同的信息结构，触控关闭按钮不小于 44px。
- 点击另一格时直接更新内容；点击关闭或按 `Esc` 收起。

### 可访问性与动效

- 格子保持键盘可聚焦，`Enter`/空格打开说明，`Esc` 关闭。
- 提示内容使用 `role="dialog"` 与明确标题关联；被查看格通过状态类和 `aria-expanded` 表达打开状态。
- 浮层出现与换位控制在 150–200ms；开启减少动态效果时取消位移和缩放动画。

## 视觉规范

- 延续现有深色与香槟金体系，风险用柔和红色、地标用紫色、奖励用金色、增益用绿色。
- 结果项采用一维列表，不使用多层嵌套卡片；主标题与本次变化拥有最高对比度，累计余额降一级显示。
- PC 提示标题约 16px、正文约 13px，保证弱化文字仍具备可读对比度。
- 提示卡最大圆角 14px，避免过度圆润；不叠加宽阴影与装饰性边框。

## 测试与验收

- PHP 测试覆盖位移后落地地标、重复地标加幸运值、地标自身幸运值效果、炸弹、冰冻和奖励格的结构化响应，以及幂等重放。
- Node 测试覆盖各反馈类型的标题、最终落点、分项收益和余额表达，不允许回退为解析长句。
- 页面测试覆盖提示卡语义结构、格子位置数据与提示内容钩子。
- 人工/静态验收覆盖视口四边自动定位、PC hover、键盘操作、手机底部面板、长名称和减少动态效果。

## 非目标

- 不改变骰子概率、格子效果、地标奖励数值或幸运值计算规则。
- 不删除历史 `result_text`，不重写旧走棋记录。
- 不新增第三方浮层或动画依赖。
