# 幸运跳棋大冒险功能说明手册

> 版本：V2.1
> 适用对象：产品、运营、客服、开发、测试及活动管理员

## 1. 系统概览

幸运跳棋大冒险是一套基于 PHP 8.2、Laravel 12 和 SQLite 的活动系统。用户通过签到、任务、充值和邀请好友获得跳棋机会，在 36 格大富翁棋盘上前进，触发奖励或事件，并参加活动排行榜。

系统主要入口：

| 页面 | 地址 | 用途 |
| --- | --- | --- |
| 登录 | `/login` | 用户及管理员登录 |
| 注册 | `/register` | 创建账号并绑定邀请码 |
| 游戏棋盘 | `/activity` | 签到、走棋、任务摘要及排行榜 |
| 幸运中心 | `/activity/center` | 任务、宝箱、收藏、领奖及消息 |
| 玩法帮助 | `/activity/help` | 新手说明、格子图例、地标图鉴、收集奖励及 FAQ |
| 运营后台 | `/admin` | 活动、用户、任务和奖励管理 |

## 2. 账号系统

### 2.1 用户注册

- 用户使用昵称、邮箱和密码注册。
- 密码至少 8 位。
- 注册时可以填写好友邀请码。
- 邀请关系一旦创建，不允许用户自行修改。
- 注册成功后自动创建活动账户和默认棋子皮肤。
- 新用户会收到欢迎站内消息。

### 2.2 用户登录

- 使用邮箱和密码登录。
- 支持“记住我”。
- 未登录访问活动页面时自动跳转到登录页。
- 登录后可以从顶部导航切换棋盘、幸运中心和管理后台。

### 2.3 用户身份

系统分为两种身份：

- 普通用户：参与活动、完成任务、走棋和领奖。
- 管理员：除普通用户功能外，还可以进入运营后台。

## 3. 跳棋机会系统

跳棋机会相当于用户的活动次数。正常走棋或冰冻解冻会消耗机会。

机会来源包括：

- 每日连续签到。
- 每日及每周任务奖励。
- 本人累计充值奖励。
- 邀请好友有效注册。
- 被邀请好友首次充值达标。
- 圈数里程碑宝箱。
- 成就奖励。
- 棋盘额外机会格。
- 管理员人工补发。

所有机会增加和扣除都会生成流水，记录：

- 发生时间。
- 业务来源。
- 增加或扣除数量。
- 变化后余额。
- 唯一业务标识。

同一业务标识不能重复发放，可以防止接口重试造成重复奖励。

## 4. 连续签到

### 4.1 签到奖励

| 连续天数 | 当天奖励 |
| --- | ---: |
| 第 1～6 天 | 5 次机会 |
| 第 7 天 | 10 次机会 |
| 第 8～13 天 | 5 次机会 |
| 第 14 天 | 10 次机会 |
| 后续 | 按 7 天周期循环 |

### 4.2 签到规则

- 每个自然日只能签到一次。
- 签到日期以活动时区为准。
- 连续签到中断后，下次从第 1 天重新计算。
- 第 7、14 天的 10 次是当天总奖励。
- 幸运中心通过 14 天日历展示签到进度和双倍奖励日。

## 5. 36 格大富翁棋盘

### 5.1 基础玩法

- 棋盘共有 36 个首尾相连的格子。
- 36 个格子沿 10×10 正方形网格的外围排布，形成紧凑的大富翁环形路线，并自适应 PC 和手机宽度。
- 用户从第 0 格起点开始。
- 每次正常走棋消耗 1 次机会。
- 点数由服务端安全随机生成，正常范围为 1～6。
- 超过第 36 格时完成一圈，剩余点数进入下一圈。
- 前端只负责播放动画，不能决定点数和奖品。
- 中央悬浮的 3D 骰子就是走棋操作入口，不再设置额外按钮；点击后骰子会翻转滚动，并在服务端返回后明确显示“本次掷出 N 点”。
- 骰子外围的 VIP、USDT、能量和地标轨道用于展示核心惊喜类型，配合剩余机会与当前位置状态，减少棋盘中央的无效留白。
- 用户当前所在格使用内外双层呼吸光圈，移动完成时会短暂强化，便于快速确认落点。
- 系统开启“减少动态效果”时，页面直接显示静态骰面和最终点数，不播放连续翻滚。

### 5.2 棋盘格类型

| 类型 | 效果 |
| --- | --- |
| 普通格 | 不产生奖励或负面效果 |
| 前进 3 格 | 立即额外前进 3 格 |
| 前进 5 格 | 立即额外前进 5 格 |
| 后退 2 格 | 后退 2 格，最低到当前圈起点 |
| 冰冻 | 用户进入冰冻状态 |
| 炸弹 | 回到当前圈起点 |
| 额外机会 | 自动获得指定数量的机会 |
| 电池 | 自动增加用户电池 |
| VIP +1 | 自动提升一级 VIP |
| 现金奖品 | 生成待领取的中奖记录 |

为了避免无限连锁，每次只触发骰子第一次落脚格的事件。事件移动后的目标格不会再次触发事件。

### 5.3 冰冻和解冻

- 用户落在冰冻格后变为冰冻状态。
- 没有解冻卡时，下次点击会消耗 1 次机会解冻，但不会移动。
- 使用解冻卡可以立即解除冰冻，不消耗跳棋机会。

### 5.4 炸弹

- 落在炸弹格后回到当前圈第 0 格。
- 已经完成的圈数不会减少。
- 防护盾或幸运卡可以抵挡炸弹。

### 5.5 请求幂等

每次走棋请求都包含唯一请求 ID。同一个请求重复提交时，系统返回第一次的结果，不会重复扣除机会或重复派奖。

### 5.6 格子图例和即时说明

棋盘将格子分为安全、地标、增益、风险和奖励五类，并使用稳定的颜色表达含义。用户可以点击手机端格子或在 PC 端悬停格子，查看格子名称、具体效果、地标解锁状态和累计到访次数。棋盘标题区域提供“格子图例”入口，顶部导航提供完整玩法帮助入口。

### 5.7 地标图鉴

原本只有文案的旅行驿站、阳光大道、彩虹街区、欢乐车站、幸运转角、星光广场、海滨大道、音乐小镇、森林公园、梦想港湾、金色大道和终点冲刺，调整为 12 个可收集地标：

- 首次到达：永久解锁对应印章，记录首次解锁时间和访问次数。
- 重复到达：增加访问次数，并额外获得 1 点幸运值。
- 地标效果：可能返还机会、提供冰冻免疫、重掷、最低点数、幸运值、随机小惊喜或防护盾。
- 图鉴累计贯穿整个活动，不随圈数重置。

收集 3、6、9、12 个不同地标时，可以在玩法帮助页开启一次性阶段宝箱。重复提交不会重复发奖。

### 5.8 新手引导与 FAQ

- 用户首次进入棋盘时显示五步引导，介绍机会来源、掷骰、地标、成长宝箱和排行榜。
- 完成或跳过后保存已读状态，后续不会反复弹出。
- 帮助页按参与活动、棋盘玩法、地标收集、道具、充值邀请、排行榜和中奖领奖分类展示 FAQ。
- 首次解锁地标时显示惊喜反馈，并引导用户继续收集地标宝箱。

## 6. 每日及每周任务

幸运中心提供完整任务进度和手动领奖功能。

默认任务包括：

| 任务 | 周期 | 条件 | 默认奖励 |
| --- | --- | --- | ---: |
| 每日签到 | 每日 | 当天签到一次 | 2 次机会 |
| 今日走 5 步 | 每日 | 完成 5 次有效走棋 | 3 次机会 |
| 完成一圈 | 每日 | 当天新增完成 1 圈 | 5 次机会 |
| 今日充值 10U | 每日 | 当天充值达到 10U | 5 次机会 |
| 每周冒险家 | 每周 | 本周完成 20 次走棋 | 10 次机会 |
| 每周召集令 | 每周 | 本周邀请 3 位好友 | 10 次机会 |

任务状态：

- 进行中：尚未达到目标。
- 可领取：目标已经完成，等待用户领取。
- 已领取：当前任务周期的奖励已经发放。

每日任务按自然日重置，每周任务按活动时区的自然周重置。同一周期只能领取一次。

## 7. 圈数里程碑宝箱

用户累计完成指定圈数后，可以开启宝箱。

默认里程碑：

| 圈数 | 宝箱 | 奖励 |
| ---: | --- | --- |
| 1 圈 | 启程宝箱 | 2 次机会 |
| 3 圈 | 青铜宝箱 | 防护盾 ×1 |
| 5 圈 | 白银宝箱 | 8 次机会 |
| 10 圈 | 黄金宝箱 | 幸运卡 ×1 |
| 20 圈 | 大师宝箱 | 20 次机会 |
| 50 圈 | 传奇宝箱 | 电池 ×10 |

每个里程碑只能领取一次。领取记录在数据库中永久保存。

## 8. 道具背包

### 8.1 道具类型

| 道具 | 功能 |
| --- | --- |
| 防护盾 | 抵挡下一次炸弹 |
| 解冻卡 | 立即解除当前冰冻状态 |
| 双倍卡 | 下一次骰子点数翻倍 |
| 重掷卡 | 下一次取两次随机点数中的较大值 |
| 幸运卡 | 免疫下一次炸弹或冰冻 |

### 8.2 使用规则

- 道具必须在幸运中心主动使用。
- 使用后从背包数量中扣除一件。
- 解冻卡立即生效。
- 其他道具生成用户活动效果，在下一次符合条件的走棋中消耗。
- 活动效果和剩余次数由服务端保存，刷新页面不会失效。
- 道具不能让前端指定中奖结果。

## 9. 成就徽章

默认成就包括：

- 初次环游：完成第一圈。
- 环游达人：累计完成 10 圈。
- 幸运降临：首次获得奖品。
- 人气召集者：邀请 5 位有效好友。
- 榜上有名：进入排行榜前 20。

达到成就条件后，系统自动解锁徽章并发放对应机会奖励。已经解锁的成就不会重复发放。

## 10. 棋子皮肤

皮肤只改变棋盘上的棋子外观，不会影响骰子点数、中奖概率或排行榜规则。

默认皮肤包括：

| 皮肤 | 图标 | 品质 | 解锁方式 |
| --- | --- | --- | --- |
| 经典跑车 | 🚗 | 普通 | 默认拥有 |
| 云端飞行家 | ✈️ | 稀有 | 完成 5 圈 |
| 黄金游艇 | 🛥️ | 史诗 | 完成 15 圈 |
| 星际探索者 | 🚀 | 传奇 | 排名进入前 10 |

用户可以在幸运中心查看已解锁皮肤并点击装备。

幸运中心根据累计圈数显示当前活动章节：

| 圈数 | 阶段 |
| --- | --- |
| 0～5 圈 | 新手旅程 |
| 6～15 圈 | 财富城市 |
| 16～30 圈 | 黄金海岸 |
| 31 圈以上 | 冠军之路 |

页面会展示当前进度和下一个阶段目标。阶段当前用于成长展示，未来可以继续关联背景、皮肤和阶段奖池。

## 12. 邀请好友中心

### 12.1 注册奖励

- 好友通过专属链接或邀请码注册。
- 邀请人获得 5 次机会。
- 同一好友只能绑定一名邀请人。
- 同一注册奖励只发放一次。

### 12.2 好友首充奖励

- 好友累计有效充值首次达到 10U。
- 邀请人获得 10 次机会。
- 每名好友只触发一次。
- 好友后续充值不再给邀请人发放首充奖励。

### 12.3 邀请进度展示

邀请中心展示：

- 成功注册人数。
- 首充达标人数。
- 累计获得机会。
- 用户邀请码。
- 每位好友的脱敏昵称。
- 好友充值进度。
- 注册奖励和首充奖励状态。

## 13. 充值奖励

- 用户每累计充值 10U 获得 10 次机会。
- 不足 10U 的余额保留到后续充值继续累计。
- 后台可以配置每日充值奖励机会限制。
- 充值订单号唯一，重复回调不能重复发放。
- 当前演示系统通过管理员人工记账模拟支付成功。
- 正式上线必须接入可信支付平台的签名回调。

## 14. 奖池和中奖播报

幸运中心展示棋盘中的主要奖品，包括：

- VIP 等级奖励。
- 电池奖励。
- 额外跳棋机会。
- USDT、PHP、VND 等资金奖励。

实时中奖播报展示：

- 脱敏用户昵称。
- 奖品名称。
- 中奖时间。

中奖播报只读取真实中奖记录，不使用虚构数据。

## 15. 中奖惊喜提示

用户落在现金、VIP 或电池等奖励格时，页面显示中奖弹窗，包括：

- 聚光和光环效果。
- 彩屑动画。
- 奖品图标悬浮动画。
- 庆祝提示音。
- 奖品名称和领取说明。

当设备开启“减少动态效果”时，系统会自动缩短或关闭动画。

## 16. 领奖中心

### 16.1 领奖方式

用户可以根据奖品类型选择：

- 数字钱包。
- 银行或本地支付方式。
- 实物配送。

用户提交钱包地址、收款信息或收件信息后，生成领奖申请。

### 16.2 领奖状态

```text
待提交 → 已提交 → 审核中 → 发放中 → 已发放
                              → 已驳回
```

VIP 和电池等奖励可以自动发放。资金和实物奖品需要运营人员完成真实发放后更新状态。

### 16.3 安全说明

- 领奖资料只允许中奖用户本人提交。
- 用户不能修改其他人的中奖记录。
- 管理后台应限制领奖资料访问权限。
- 正式环境需要按照当地隐私法规保存和清理敏感资料。

## 17. 排行榜

### 17.1 总进度榜

依次按照以下条件排名：

1. 完成圈数更多。
2. 当前格子更大。
3. 更早到达当前进度。
4. 用户 ID 更小。

### 17.2 今日活跃榜

按照当天有效走棋次数排名。

### 17.3 邀请榜

按照成功邀请的有效好友数量排名。

### 17.4 最终奖励

| 排名 | 奖励 |
| --- | --- |
| 第 1 名 | iPhone 16 Pro |
| 第 2 名 | 500 USDT |
| 第 3 名 | 400 USDT |
| 第 4 名 | 300 USDT |
| 第 5 名 | 200 USDT |
| 第 6～10 名 | 每人 100 USDT |

活动结束后，管理员应先冻结活动、完成风控审核，再确认最终排名。

## 18. 消息中心

系统可以生成以下消息：

- 新用户欢迎消息。
- 邀请好友注册成功。
- 好友首次充值达标。
- 任务、里程碑和成就奖励到账。
- 用户获得奖品。
- 奖品领取和发放提示。
- 系统及活动公告。

首页会显示未读消息数量。用户进入幸运中心后可以查看消息，并执行“全部已读”。

## 19. 运营后台

### 19.1 数据概览

后台展示：

- 参与用户数。
- 累计走棋次数。
- 中奖记录数。
- 待处理奖励数。

### 19.2 活动配置

管理员可以设置：

- 活动开始时间。
- 活动结束时间。
- 每日充值奖励机会限制。
- 草稿、进行中、冻结结算和已结束状态。

非“进行中”状态或不在活动时间范围内时，用户不能继续走棋。

### 19.3 充值记账

- 选择用户并录入到账 USDT 金额。
- 系统计算本人累计充值奖励。
- 好友首次累计达到 10U 时，自动给邀请人发放一次奖励。
- 正式生产应由支付平台回调替代人工记账。

### 19.4 机会调整

- 管理员可以补发或扣回用户机会。
- 必须填写调整原因。
- 调整结果写入用户机会流水。
- 操作写入管理员审计日志。

### 19.5 任务管理

- 管理员可以开启或停用每日、每周任务。
- 停用后用户端不再展示该任务。
- 已经产生的历史任务记录不会被删除。

### 19.6 用户进度

后台可以查看：

- 用户 ID、昵称和邮箱。
- 剩余机会。
- 完成圈数和当前格子。
- VIP 等级。
- 电池数量。

### 19.7 领奖审核

管理员可以查看用户提交的领奖资料，并更新为：

- 已提交。
- 审核中。
- 发放中。
- 已发放。
- 已驳回。

### 19.8 管理员审计日志

审计日志记录：

- 管理员身份。
- 操作时间。
- 操作名称。
- 操作对象。
- 操作数据。
- IP 地址。

## 20. 活动状态

| 状态 | 说明 |
| --- | --- |
| 草稿 | 活动配置阶段，不允许正式走棋 |
| 进行中 | 用户可以正常签到、任务和走棋 |
| 冻结结算 | 停止走棋，用于审核和生成最终榜单 |
| 已结束 | 活动正式结束，仅保留记录和领奖流程 |

## 21. 响应式设计

### PC 端

- 棋盘和运营信息采用双栏布局。
- 任务、排行榜和幸运中心入口位于棋盘右侧。
- 记录区域位于棋盘下方。

### 手机端

- 棋盘等比缩放并完整显示，不需要横向滚动。
- 状态数据压缩为四列。
- 任务、排行榜和记录改为纵向布局。
- 幸运中心导航支持横向滑动。
- 表格内容在卡片内部滚动。

## 22. 安全和一致性

- 所有写操作均受登录鉴权和 CSRF 保护。
- 管理后台操作验证管理员身份。
- 走棋、扣次数、移动和派奖使用数据库事务。
- 走棋请求使用唯一请求 ID 防止重复操作。
- 充值订单号和奖励业务标识唯一。
- 点数和事件结果只由服务端计算。
- 密码使用 Laravel 安全哈希保存。
- SQLite 采用 WAL、短事务和备份策略。

## 23. 第三方系统接入边界

以下功能已经完成站内流程，但正式上线需要外部平台：

| 功能 | 需要接入 |
| --- | --- |
| 真实充值 | 支付平台签名回调 |
| USDT 发放 | 钱包或资金系统 API |
| PHP/VND 发放 | 本地支付渠道 |
| 邮件通知 | SMTP 或邮件服务商 |
| 短信通知 | 短信服务商 |
| 实物物流 | 物流或人工发货流程 |

没有配置第三方密钥时，系统不会伪造真实支付或钱包到账，只提供人工记账、审核和状态追踪。
