# 幸运跳棋大冒险操作手册

## 1. 演示账号

| 身份 | 邮箱 | 初始密码 | 入口 |
| --- | --- | --- | --- |
| 普通用户 | `demo@example.com` | `Demo123!` | `/login` |
| 管理员 | `admin@example.com` | `Admin123!` | `/admin` |

正式上线前必须修改或删除演示账号，并使用高强度管理员密码。

## 2. 用户操作

### 幸运中心

登录后可从顶部导航或棋盘右侧进入“幸运中心”，包含：

- 每日、每周任务及奖励领取。
- 14 天连续签到日历。
- 圈数里程碑宝箱。
- 邀请好友注册和首充达标进度。
- 防护盾、解冻卡、双倍卡、重掷卡和幸运卡。
- 成就徽章及棋子皮肤收藏。
- 全站中奖播报和个人领奖资料提交。
- 总进度榜、今日活跃榜和邀请榜。
- 奖励、邀请、中奖及系统消息。

任务和里程碑达到条件后需要点击领取。同一周期或同一里程碑只能领取一次。

### 玩法说明、地标图鉴和 FAQ

登录后点击顶部“玩法”，或点击棋盘的“格子图例”进入说明：

- 首次进入棋盘会看到五步新手引导，可以完成或跳过。
- PC 端将鼠标悬停到格子，手机端点击格子，可以查看该格的具体效果。
- 紫色格为地标格，首次到达会解锁印章；重复到达会增加访问次数并获得幸运值。
- 在 `/activity/help` 可查看 12 个地标的效果和个人解锁进度。
- 收集 3、6、9、12 个不同地标后，在“地标收集宝箱”点击开启奖励；每档只能领取一次。
- 常见规则按类别收录在 FAQ 中，点击问题即可展开答案。

### 注册和邀请

1. 打开注册页，填写昵称、邮箱和至少 8 位密码。
2. 通过好友邀请链接进入时，邀请码会自动填写。
3. 注册成功后自动登录并进入活动页面。
4. 有效邀请注册后，邀请人获得 5 次机会。

用户可以在“今日任务”的“邀请好友”中复制专属邀请链接。

### 签到

1. 登录活动页面。
2. 点击“签到”。
3. 普通连续签到获得 5 次；连续第 7、14、21 天获得 10 次。
4. 断签后，下次从连续第 1 天重新计算。

### 跳棋

1. 查看页面顶部剩余机会。
2. 点击棋盘中央“立即跳棋”。
3. 系统随机生成 1～6 点并移动棋子。
4. 落在奖励或事件格后，页面显示处理结果。
5. 冰冻状态下，下次点击只消耗 1 次机会解冻，不移动。

首次解锁地标会显示“新地标已解锁”惊喜提示。普通安全格不会发奖，但会明确提示“安全抵达”，用于调节事件密度和游戏节奏。

前进、后退、炸弹、冰冻、额外机会和奖品均由服务端计算，刷新或重复点击不会改变已经生成的结果。

道具规则：

- 防护盾抵挡下一次炸弹。
- 解冻卡立即解除冰冻，不消耗跳棋机会。
- 双倍卡使下一次骰子步数翻倍。
- 重掷卡使下一次使用两次随机点数中的较大值。
- 幸运卡免疫下一次炸弹或冰冻。

道具生效、失效时间和剩余数量均由服务端保存。

### 查看记录和榜单

- “机会明细”查看签到、邀请、充值、跳棋和解冻的机会变化。
- “中奖列表”查看奖品及发放状态。
- 右侧或棋盘下方查看实时排行榜前 20 名。
- 排名依次比较完成圈数、当前格子和到达时间。

## 3. 管理员操作

### 活动运营控制台

管理员可以：

- 调整活动开始、结束时间及活动状态。
- 设置每日充值奖励机会限制。
- 开启或关闭每日、每周任务。
- 人工补发或扣回用户机会，并填写原因。
- 查看用户、走棋、中奖和待处理奖励数据。
- 审核领奖资料并更新为审核中、发放中、已发放或驳回。
- 查看管理员操作审计日志。

活动状态设置为“冻结结算”或“已结束”后，用户不能继续走棋，但仍可以查看活动数据和领奖状态。

### 充值记账

当前首版使用人工充值记账模拟支付成功回调：

1. 管理员登录后进入 `/admin`。
2. 在“人工充值记账”中选择用户并输入 USDT 金额。
3. 点击“确认到账”。
4. 用户每累计达到 10U 自动获得 10 次机会。
5. 被邀请好友首次累计达到 10U 时，邀请人获得一次 10 次机会；以后不再重复发放。

正式生产环境应将该流程替换为支付平台签名回调，不能相信用户前端提交的充值金额。

### 中奖发放

1. 在后台“中奖发放”中找到待发放记录。
2. 财务或运营完成真实发放。
3. 点击“标记已发放”。
4. 状态变为 `issued`，用户端显示“已发放”。

标记操作不能代替真实钱包或人工派奖。

资金或实物奖品应先由用户在幸运中心提交钱包、本地收款账户或收件信息。管理员完成核验和真实发放后，再更新系统状态。敏感领奖资料应限制后台访问权限并按当地隐私规定保存。

### 用户进度

后台可以查看用户剩余机会、完成圈数、当前格子、VIP 和电池数量，用于客服核对和活动风控。

## 4. 日常检查

- 每天检查待发放中奖记录和失败的充值回调。
- 检查 `storage/logs/laravel.log` 中的异常。
- 检查磁盘空间和 SQLite 备份是否成功。
- 关注排名前 20 用户是否存在相同设备、IP 或异常充值。
- 活动结束后停止继续跳棋，冻结最终榜单并完成风控审核后派奖。

## 5. 常见问题

### 页面显示 500

检查 `.env`、文件权限、PHP SQLite 扩展和 `storage/logs/laravel.log`。

### 提示 `could not find driver`

安装 SQLite 扩展并重启 PHP-FPM：

```bash
sudo apt install php8.2-sqlite3
sudo systemctl restart php8.2-fpm
```

### 提示 `database is locked`

确认已开启 WAL，缩短写事务，并检查是否存在长时间运行的写任务。低访问量下偶发锁等待可以通过 `busy_timeout` 重试；持续出现时应迁移到 MySQL 或 PostgreSQL。

### 修改代码后页面没有变化

```bash
php artisan optimize:clear
npm run build
```
