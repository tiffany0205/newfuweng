# 幸运跳棋大冒险操作手册

## 1. 演示账号

| 身份 | 邮箱 | 初始密码 | 入口 |
| --- | --- | --- | --- |
| 普通用户 | `demo@example.com` | `Demo123!` | `/login` |
| 管理员 | `admin@example.com` | `Admin123!` | `/admin` |

正式上线前必须修改或删除演示账号，并使用高强度管理员密码。

## 2. 用户操作

### 注册和邀请

1. 打开注册页，填写昵称、邮箱和至少 8 位密码。
2. 通过好友邀请链接进入时，邀请码会自动填写。
3. 注册成功后自动登录并进入活动页面。
4. 有效邀请注册后，邀请人获得 5 次机会。

用户可以在“今日任务”的“邀请好友”中复制专属邀请链接。

### 签到

1. 登录活动页面。
2. 点击“签到”。
3. 普通连续签到获得 5 次；连续第 7、14、21 天获得 10 次。
4. 断签后，下次从连续第 1 天重新计算。

### 跳棋

1. 查看页面顶部剩余机会。
2. 点击棋盘中央“立即跳棋”。
3. 系统随机生成 1～6 点并移动棋子。
4. 落在奖励或事件格后，页面显示处理结果。
5. 冰冻状态下，下次点击只消耗 1 次机会解冻，不移动。

前进、后退、炸弹、冰冻、额外机会和奖品均由服务端计算，刷新或重复点击不会改变已经生成的结果。

### 查看记录和榜单

- “机会明细”查看签到、邀请、充值、跳棋和解冻的机会变化。
- “中奖列表”查看奖品及发放状态。
- 右侧或棋盘下方查看实时排行榜前 20 名。
- 排名依次比较完成圈数、当前格子和到达时间。

## 3. 管理员操作

### 充值记账

当前首版使用人工充值记账模拟支付成功回调：

1. 管理员登录后进入 `/admin`。
2. 在“人工充值记账”中选择用户并输入 USDT 金额。
3. 点击“确认到账”。
4. 用户每累计达到 10U 自动获得 10 次机会。
5. 被邀请好友首次累计达到 10U 时，邀请人获得一次 10 次机会；以后不再重复发放。

正式生产环境应将该流程替换为支付平台签名回调，不能相信用户前端提交的充值金额。

### 中奖发放

1. 在后台“中奖发放”中找到待发放记录。
2. 财务或运营完成真实发放。
3. 点击“标记已发放”。
4. 状态变为 `issued`，用户端显示“已发放”。

标记操作不能代替真实钱包或人工派奖。

### 用户进度

后台可以查看用户剩余机会、完成圈数、当前格子、VIP 和电池数量，用于客服核对和活动风控。

## 4. 日常检查

- 每天检查待发放中奖记录和失败的充值回调。
- 检查 `storage/logs/laravel.log` 中的异常。
- 检查磁盘空间和 SQLite 备份是否成功。
- 关注排名前 20 用户是否存在相同设备、IP 或异常充值。
- 活动结束后停止继续跳棋，冻结最终榜单并完成风控审核后派奖。

## 5. 常见问题

### 页面显示 500

检查 `.env`、文件权限、PHP SQLite 扩展和 `storage/logs/laravel.log`。

### 提示 `could not find driver`

安装 SQLite 扩展并重启 PHP-FPM：

```bash
sudo apt install php8.2-sqlite3
sudo systemctl restart php8.2-fpm
```

### 提示 `database is locked`

确认已开启 WAL，缩短写事务，并检查是否存在长时间运行的写任务。低访问量下偶发锁等待可以通过 `busy_timeout` 重试；持续出现时应迁移到 MySQL 或 PostgreSQL。

### 修改代码后页面没有变化

```bash
php artisan optimize:clear
npm run build
```
